# bag_assistant_project/ ИИ-ассистент тех.поддержки по работе с багами 
## Описание текущего процесса
Операторы технической поддержки при каждом обращении клиента с проблемой вынуждены вручную просматривать актуальные открытые и закрытые баги, размещенные во внутренних ресурсах в формате pdf, excel, google sheets, а также в ТГ канале. Они пытаются понять, подходит ли проблема клиента под один из багов, чтобы предложить наиболее релевантное решение. 
## Проблема
Большие затраты времени операторов на поиск подходящих багов и высокая вероятность пропуска подходящего бага из-за человеческого фактора и неудобных источников инфо. Приводят к увеличению среднего времени обработки обращения клиента (AHT) и росту повторных обращений клиентов (FCR), что способствует увеличению общих затрат на тех.поддержку.
## Решение
Бот ассистент тех.поддержки по работе с багами на основе запроса пользователя, которое в него перенаправляет оператор (либо подтягивается автоматом в идеале), анализирует баги, кратко описанные в pdf и google sheets, и выбирает из них, под какие баги может подходить проблема клиента.

## Стратегия
Исходя из расчетов стоимости - облачное решение для MVP подойдет лучше, тк дешевле локального - 3600$ разово+ опеационные расходы 300$/мес. При этом если стартовый вариант принесет реальную экномию бюджета в долговсрочной перспективе, а также с учетом масштабирования и требований безопасности данных клиентов, использования интеграций с внутренними базами данных, которые будут отправляться в Input, я бы перешла на локальный вариант.
